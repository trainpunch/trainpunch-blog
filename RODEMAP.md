# TrainPunch Blog ROADMAP
GitHub Pages + 静的サイト（Hugo 想定）で  
**「一般向けウェイトトレーニング／自重トレーニング特化ブログ」** を作るためのロードマップ。

※実装時のメモ用なので、細かい手順は別途 README に切り出してもOK。

---

## 0. ゴール定義

- [ ] ブログの目的と言語・対象を決める
  - 想定読者：
    - 20〜40代くらいの一般の人
    - ダイエット・ボディメイク・健康目的で筋トレを始めたい／続けたい人
    - ジム通い or 自宅トレ（自重＋ダンベル程度）
- [ ] ブログの役割を決める
  - [ ] 「難しいこと抜きで筋トレを習慣化したい人」のガイド
  - [ ] 自重／簡易器具メインで、現実的に続けられるメニューを提案
  - [ ] AdSense を使った収益化
  - [ ] 将来 TrainPunch への導線（メニュー連携・記録の提案）
- [ ] 1年後の最低ラインを決める
  - 例：
    - 月間 1万PV
    - 記事 30〜50 本
    - 「家トレ」「ジム初心者」系キーワードでいくつか検索1ページ目表示

---

## 1. リポジトリ設計 & GitHub Pages 方針

- [ ] ブログ用リポジトリ名を決める
  - 例：`trainpunch-blog`
- [ ] 公開 URL 方針を決める
  - [ ] `https://trainpunch.github.io/trainpunch-blog/`（サブディレクトリ）
  - 既存 TrainPunch PWA サイトからナビゲーションリンクを張る構成にする
- [ ] GitHub Pages のデプロイブランチ/フォルダの方針を決める
  - 例：`main` ブランチの `docs/` フォルダを公開する

```text
【想定ディレクトリ構成】
- main ブランチ
  - /docs       ← ビルド済み静的ファイル（GitHub Pages 公開用）
  - /content    ← 記事（Markdown）
  - /layouts    ← テンプレート
  - /static     ← 画像など
  - config.toml / package.json など設定ファイル類
